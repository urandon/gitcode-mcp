# 018-snapshot-diff-task-1-snapshotstore-stored-diff validation scenarios

- `018-snapshot-diff-task-1-snapshotstore-stored-diff-scenario-1`: A developer runs `gitcode-mcp index --repo fixture-repo`, captures a stored snapshot ID, then runs `gitcode-mcp export-snapshot --repo fixture-repo --id <id> --format json`; the CLI response includes the requested `repo_id`, `snapshot_id`, repo-scoped source summaries, stored chunks from `snapshot_chunks`, byte/line citation ranges, and deterministic machine-readable warnings, with executable evidence from a CLI integration test over fixture snapshots.
- `018-snapshot-diff-task-1-snapshotstore-stored-diff-scenario-2`: A fixture test mutates one cached record after indexing without rebuilding chunks, then creates and exports a snapshot; `export-snapshot` reports stored `stale_index` for the affected source, while a fixture source with no chunks reports stored `missing_index`, with evidence from the same CLI test suite.
- `018-snapshot-diff-task-1-snapshotstore-stored-diff-scenario-3`: A developer runs `gitcode-mcp diff-snapshot --repo fixture-repo --base-id <valid-base> --head-id <valid-head> --format json`; the CLI returns deterministic `added`, `removed`, and `modified` entries ordered by the stored comparison contract, including `citation_range_changed` for range-only chunk changes, and service/CLI tests prove current cache mutations after snapshot creation do not affect the diff.
- `018-snapshot-diff-task-1-snapshotstore-stored-diff-scenario-4`: A developer runs `gitcode-mcp diff-snapshot --repo fixture-repo --base-id missing --head-id <valid-head>`; the CLI returns typed not-found for `base_id` and does not produce `changed:false` or a current/current diff, with executable evidence from CLI and `internal/snapshot` tests.
